## [1.0.0] - 2026-06-28

### Added
- ✨ Основной анализатор git репозитория
  - Анализ последних N коммитов
  - Подсчёт строк кода по авторству
  - Определение AI-авторства по ключевым словам
  - Поддержка co-authored commits
  
- 📊 Вычисление Vibe Index
  - Формула с весами (60% код, 40% коммиты)
  - Масштабирование 0-10 баллов
  - Цветовая схема автоматического выбора
  
- 🎨 Генерация badge
  - shields.io интеграция
  - Поддержка разных стилей
  - Markdown и HTML синтаксис
  
- ⚙️ Валидация входных параметров
  - Проверка всех inputs
  - Полезные сообщения об ошибках
  - Предотвращение инъекций
  
- 🛡️ Качественный контроль
  - Assert mode для проверки диапазона
  - Падение workflow при нарушении
  
- 📚 Документация
  - README с полным API
  - QUICKSTART для быстрого старта
  - EXAMPLES с 10+ вариантами
  - DEVELOPMENT гайд
  - CONVENTIONS для разработчиков
  - STRUCTURE обзор архитектуры
  - CLAUDE для AI разработчиков
  
- 🧪 Тестирование
  - Юнит тесты для вычислений
  - Проверка edge cases
  - Валидация входов
  
- 📝 GitHub Actions примеры
  - vibe-index-badge.yml — базовый анализ
  - vibe-index-assert.yml — с проверкой

### Initial Release
- Первый публичный релиз Vibe Index GitHub Action
- Готово к использованию в GitHub Actions workflows

## Структура версий

### Major (v2.0.0, v3.0.0, ...)
Несовместимые изменения:
- Изменение формулы Vibe Index
- Удаление параметра input
- Переименование output
- Breaking changes в API

### Minor (v1.1.0, v1.2.0, ...)
Новые функции (обратно совместимые):
- Новый параметр input
- Новый output
- Новая фича
- Улучшение производительности

### Patch (v1.0.1, v1.0.2, ...)
Исправления:
- Исправление ошибок
- Улучшения документации
- Оптимизация
- Исправления безопасности

## Как работать с релизами

### Создание новой версии

1. **Обновить CHANGELOG.md**
   ```markdown
   ## [1.1.0] - 2026-07-15
   
   ### Added
   - Новая функция
   ```

2. **Создать git tag**
   ```bash
   git tag v1.1.0
   git push origin v1.1.0
   ```

3. **GitHub автоматически создаст Release** с Notes из CHANGELOG

### Версионирование

```
v1.0.0  — первый релиз
v1.0.1  — исправление ошибки
v1.1.0  — новая функция
v2.0.0  — breaking change
```

## Как внести изменение

1. **Сделай изменение в коде**
2. **Создай commit** с правильным типом:
   ```bash
   git commit -m "feat: добавить новую функцию"
   git commit -m "fix: исправить ошибку"
   ```
3. **Обнови CHANGELOG.md** в секции `[Unreleased]`
4. **Когда готово к релизу:**
   - Переименуй `[Unreleased]` на `[1.1.0] - YYYY-MM-DD`
   - Создай git tag
   - Пушни в GitHub
